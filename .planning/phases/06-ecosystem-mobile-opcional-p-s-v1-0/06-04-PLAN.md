<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| agentskills-install skill → local filesystem | Skill installs a file from an external URL to skills/; path traversal or malicious content is a risk |
| agentskills-publish → public git repo | Skill may include secrets if user publishes a skill with API keys hardcoded |
| channel-scaffold generated skill → daemon HTTP | Community-built channels inherit the daemon's trust surface; must implement CR-03 auth |
| ClawHub skills → Bastion runtime | Imported skills from ClawHub may have unvalidated triggers or missing privacy_tier |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-04-01 | Tampering | agentskills-install writes file with path traversal in name | mitigate | install skill documents: validate name; reject names containing `..`, `/`, `\`. Confirmation step before write. |
| T-06-04-02 | Information Disclosure | agentskills-publish exposes skill with hardcoded secrets | mitigate | Publish skill has explicit security note: check for secrets before publish; check privacy_tier = cloud-ok only. |
| T-06-04-03 | Elevation of Privilege | community channel skill omits OwnerMap.resolve(token) auth | mitigate | channel-scaffold documents CR-03 requirement as a mandatory step. Community devs who skip it are explicitly warned in the scaffold doc. |
| T-06-04-04 | Spoofing | ClawHub skill with missing privacy_tier defaults to unrestricted egress | mitigate | ClawHub migration doc notes: missing privacy_tier in metadata → Bastion defaults to CloudOk; document this; advise adding explicit tier. |
| T-06-04-05 | Tampering | Installed skill SKILL.md contains malicious instructions that modify system behavior | accept | SKILL.md is a definition file; Bastion reads it but does not exec code from it. The skill-writer pattern (read → confirm → write) is the only execution path. Risk is low for skill definitions. |
</threat_model>
